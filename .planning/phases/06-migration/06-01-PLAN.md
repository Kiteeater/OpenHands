<objective>
Migrate auth.py and oauth_device.py analytics call sites to use the Phase 5 Foundation patterns: resolve_context for user/consent/org resolution, identify_user for identity tracking, and typed event methods for event capture.

Purpose: These two files contain the most complex analytics code — duplicated identity blocks (set_person_properties + group_identify loops) that Phase 5's identify_user was specifically designed to replace. Migrating them eliminates the largest source of analytics code duplication (REFAC-06) and the most inline user lookup / raw property dict patterns (REFAC-02, REFAC-04).

Output: auth.py and oauth_device.py using resolve_context, identify_user, track_user_signed_up, and track_user_logged_in instead of inline patterns.
</objective>

**1. User signed up block (lines ~257-286 in keycloak_callback):**

# Use resolve_context for the AnalyticsContext (note: user object is already loaded above as `user`)

consented = user.user_consents_to_analytics is True

**2. Identity block (lines ~436-498 in keycloak_callback):**

Current code does inline `set_person_properties()` with 6 fields, then loads all user orgs, loops with `group_identify()` for each. This is EXACTLY the pattern identify_user was built to replace.

Replace the entire identity block (from `analytics = get_analytics_service()` through the group_identify loop) with:
```python
analytics = get_analytics_service()
if analytics:
    consented = user.user_consents_to_analytics is True
    org_id_str = str(user.current_org_id) if user.current_org_id else None

    # Load org data for identify_user
    from storage.org_store import OrgStore
    current_org = await OrgStore.get_org_by_id(user.current_org_id) if user.current_org_id else None
    org_member_ids = [om.org_id for om in user.org_members] if user.org_members else []
    user_orgs = await _get_user_orgs_with_data(user_id, org_member_ids)

    from storage.org_member_store import OrgMemberStore
    orgs_data = []
    for org in user_orgs:
        try:
            member_count = await OrgMemberStore.get_org_members_count(org_id=org.id)
        except Exception:
            logger.exception('auth:identify_user:member_count_failed', extra={'user_id': user_id, 'org_id': str(org.id)})
            member_count = None
        orgs_data.append({'id': str(org.id), 'name': org.name, 'member_count': member_count})

    analytics.identify_user(
        distinct_id=user_id,
        consented=consented,
        email=email,
        org_id=org_id_str,
        org_name=current_org.name if current_org else None,
        idp=idp,
        orgs=orgs_data,
    )

    analytics.track_user_logged_in(
        distinct_id=user_id,
        idp=idp,
        org_id=org_id_str,
        consented=consented,
    )
```

  <action>
Migrate the analytics block in `enterprise/server/routes/oauth_device.py` `device_verification_authenticated` (lines ~316-360) to use Phase 5 Foundation patterns.

**Current code** does inline:
1. `UserStore.get_user_by_id(user_id)` for consent/org resolution
2. `OrgStore.get_org_by_id(user.current_org_id)` for org name
3. `analytics.set_person_properties(...)` with org_id, org_name, idp
4. `analytics.capture(event='user logged in', properties={'idp': 'device_auth'})`

**Replace with** resolve_context + identify_user + track_user_logged_in:

```python
# Server-side identity tracking for device auth flow
analytics = get_analytics_service()
if analytics:
    try:
        from openhands.analytics import resolve_context

        ctx = await resolve_context(user_id)

        # identify_user for person properties + org group (no orgs list needed here,
        # identify_user handles the person-level properties)
        from storage.org_store import OrgStore
        current_org = await OrgStore.get_org_by_id(ctx.user.current_org_id) if ctx.user and ctx.user.current_org_id else None

        analytics.identify_user(
            distinct_id=user_id,
            consented=ctx.consented,
            org_id=ctx.org_id,
            org_name=current_org.name if current_org else None,
            idp='device_auth',
        )

        analytics.track_user_logged_in(
            distinct_id=user_id,
            idp='device_auth',
            org_id=ctx.org_id,
            consented=ctx.consented,
        )
    except Exception:
        logger.exception(
            'oauth_device:analytics:failed',
            extra={'user_id': user_id},
        )
```

**Update imports:**

The file currently imports `from openhands.analytics import get_analytics_service`. Keep this. The `resolve_context` import can be at the top of the file for clarity:

from openhands.analytics import get_analytics_service, resolve_context

Remove the now-unused `from storage.user_store import UserStore` import from within the analytics block (note: the top-level import of `DeviceCodeStore` at line 10 is from `storage.device_code_store`, not user_store).

- Error isolation is maintained by the outer try/except (resolve_context also has its own)
